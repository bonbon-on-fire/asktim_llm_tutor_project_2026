### `run_ui_mini` restore behavior (requested change)

When restoring at **turn X**:

- Keep all **student + tutor** exchanges through **turn X − 1** (complete).
- For **turn X**, keep **only the student message** (drop the tutor reply for that turn if present).
- **Continue** from there with the **tutor responding first** for turn X, then alternate as usual.

### Future tutor structures (options)

- **Internal state**: maintain a variable for the student’s **current problem/challenge**; once resolved, tutor **backs off** until the student brings a new problem.
- **Two-layer agentic tutor**: bottom layer behaves as today; **top layer** has rubric access, reviews tutor output **before** it reaches the student, sends feedback to bottom layer and **re-runs** if needed — cap at **one** re-run per turn.
- **Performance**: check whether dropping **pedagogical reasoning** speeds the two-layer design.
- **Correctness**: verify `run_ui_mini` does **not** feed prior transcripts’ pedagogical reasoning into the next tutor run as input (double-check implementation).

### Prompt versions and lightweight judging idea

- For now, **three tutor prompt versions** are enough; expect more prompt tweaks after the **next** meeting.
- **Future sanity check**: a rubric or judge that **compares** mini outputs rather than absolute scores — e.g. random transcript, random turn, re-run **only that turn** via `run_ui_mini` with the new prompt, then **compare** turns (comparison-based, not full-rubric grading).
