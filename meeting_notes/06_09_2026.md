### Testing methodology

- For new course testing, the team will use a combination of:
  - Student persona simulations.
  - Manual hand testing.
  - The judge/evaluator framework used previously.

- Student personas were previously too repetitive, but they are still useful as a sanity check for endpoint cases.
- The team should improve the student persona approach by reviewing recent papers on using LLMs to simulate students.

### Tutor context expansion

- The tutor should be expanded to support richer course context.
- Priority context improvements include:
  - Accepting images as input.
  - Producing or reasoning over image-based outputs where relevant.
  - Adding lecture transcripts as part of the tutor context.

- These improvements are especially important for courses where diagrams, visuals, or lecture-specific explanations matter.

### Internal tooling and data review

- A lower-priority but useful improvement is to create a small interface for reviewing real AskTIM data.
- This would avoid needing to inspect conversations manually through Railway.
- The interface would make it easier to review, share, and analyze real student interactions.
