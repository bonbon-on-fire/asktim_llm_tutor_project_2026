## Notes

### SC2x context setup is in place

- Course content added and reformatted to match the other courses' conventions: `course.txt`, `syllabus.txt`, and Weeks 1–3 graded assignments as `exercise_01..03` (Option 1 — all of a week's problems as one exercise context).
- Practice problems are now a distinct, selectable content kind (`practices/practice_<NN>.txt`, surfaced in the Create-context wizard). Scraped 41 practice problems for Weeks 1–4 and 7–10.
- Coverage note: only Weeks 1–3 graded assignments are available on the Audit track; Weeks 4 and 7–10 graded assignments and the exams are Verified-track-only (paywalled) or not yet released.
- Lecture transcripts scraped from MIT Learn (login-gated) via the courseware transcript handler: 156 per-video transcripts (Weeks 0–10) plus the SC2x section of the Key Concept Document — ~374k tokens total.

### RAG framework is built and working

- Built a per-course RAG index for SC2x from the local lecture transcripts (766 chunks, `text-embedding-3-small`). Retrieval was verified end-to-end (e.g., a facility-location question pulls the right lecture chunks by meaning).
- The Sandbox "Use RAG for course context" toggle is now active for SC2x, since the course has an index.
- This validates the scalable-context approach from the 06/26 meeting: the full lecture corpus is far too large to put in every turn (~374k tokens), so RAG retrieves only the relevant chunks per student turn.

### First simulated-test results

- Ran 108 simulated conversations: 3 exercises + 3 practice problems × 18 student personas (chaotic / clueless / cooperative, 6 each) × 10 turns, tutor_05 on Claude, with a lecture-free context (course + syllabus + the problem).
- Graded all 108 with the Claude judge (judge_08 / rubric_08): mean 37.1/40, 0 failures.
- Scores by persona type: cooperative 39.4, chaotic 37.0, clueless 35.0 (out of 40).
- Where points are lost: the tutor is weakest on dialogue quality with clueless students (~79%) and pedagogy with clueless students (~88%); communication is strong across all persona types (99–100%). Exercises and practice problems score almost identically.
- Charts for all of the above are checked in under `visualization/outputs/sc2x/`.

### Production bug found and fixed during testing

- Testing surfaced a real bug: the tutor injects the assignment text into its system prompt via a regex replacement, which mis-interpreted backslashes (e.g., LaTeX like `\sum`, `\leq`) and crashed with "bad escape \s". This would have broken the live tutor on any math-heavy assignment.
- Fixed in both `tutor/run_tutor.py` and the Sandbox custom-prompt renderer (use a replacement function so text is inserted literally).

### Wizard / UX updates

- Course-description: replaced the "No course description" dropdown option with an "Include course description" checkbox toggle (mirrors the RAG toggle), so course identity and description inclusion are independent.
- Exercise step now lists exercises and practice problems as one flat list (no group headers).

## Decisions

- Confirmed Option 1 for exercise context (one tutor per week knowing all of that week's problems), now implemented for SC2x.
- Use RAG for the large lecture corpus; keep the small, essential context (exercise/practice text, course, syllabus) injected verbatim.
- For the first simulation round, run a lecture-free context to keep cost/latency manageable and establish a baseline before layering RAG in.
- Keep practice problems as a first-class, separately-labeled content kind.
