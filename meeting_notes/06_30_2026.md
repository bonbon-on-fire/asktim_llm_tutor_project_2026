## Notes

### CTL.SC2x context setup (recap)

- The course content is in and reformatted to match the other courses: course description, syllabus, and the Weeks 1-3 graded assignments set up as one exercise context per week (Option 1, where the tutor for a week knows all of that week's problems).
- Practice problems are now a separate, selectable context type in the Create-context wizard, alongside the graded exercises. We scraped 41 practice problems covering Weeks 1-4 and 7-10.
- We do not have full access to all the exercises: only the Weeks 1-3 graded assignments are available on the Audit track. The Weeks 4 and 7-10 graded assignments and the exams are Verified-track-only (paid) or not yet released.
- We scraped the lecture transcripts from the login-gated MIT Learn site: 156 per-video transcripts across Weeks 0-10 plus the course's Key Concept Document. Altogether this is about 374,000 tokens of lecture material.
- The RAG retrieval framework is built and working for this course: it pulls only the handful of relevant lecture chunks per student turn instead of the full transcripts, and the Sandbox now has a toggle to turn RAG-based course context on.

### First round of simulated-conversation test results

- We ran 108 simulated conversations: 3 exercises and 3 practice problems, each run once against all 18 student personas (chaotic, clueless, and cooperative, six versions each), at 10 turns per conversation, using the Claude tutor.
- All 108 were graded by the Claude judge, with no failures and a mean score of 37.1 out of 40.
- By persona type, the tutor scored highest with cooperative students (39.4), then chaotic (37.0), then clueless (35.0). Exercises and practice problems scored almost the same.
- Charts of all of this were generated and checked in.

### Where the tutor is weakest and whether it fits the course

- The tutor is weakest with clueless students, mostly on keeping a productive dialogue going (about 79%) and on pedagogy (about 88%). Communication is strong across all persona types (99-100%).
- Looking at the lowest-scoring transcripts (for example, a chaotic student on exercise 1 scoring 31 out of 40), the tutor correctly refuses to hand over answers but loses points for repeating the refusal without reframing and for not pivoting to productive tutoring sooner, not for being wrong.
- Important caveat for this quantitative course: every problem is auto-graded (numeric entry or multiple choice / select-all), with no free-response. The current rubric grades pedagogy, dialogue, and communication, but not whether the tutor's guidance actually reaches the correct answer, so the scores can look strong while math correctness is still untested.
- Testing surfaced and fixed a real bug where math notation in an assignment was crashing the tutor's prompt setup; this is now fixed in the main tutor and the Sandbox.

### Access, context additions, and priorities

- We need an updated list from Matt of the context and access we still need, since we do not have full access to all the course exercises yet.
- Reviewing some of the transcripts, we noticed that a few problems require information from the lecture to solve, so we will need to add lecture content as part of the tutor's context. This is what the RAG framework is for, and it should also be wired into the Sandbox.
- The way context is displayed on the dashboard website has a formatting issue that needs to be fixed.
- The single most important thing for this tutor is that it does not leak answers. This is a very serious course, so preventing answer leakage takes priority over every other improvement.

## Decisions

- Use Option 1 for exercise context: one tutor per week that knows all of that week's problems. This is now set up for SC2x.
- Use RAG for the large lecture material, and keep the small, essential context (the problem, course description, and syllabus) in the prompt directly.
- Run the first round of simulations without lecture context to get a baseline before adding RAG.
- Keep practice problems as a separate, clearly labeled context type.
- Make answer-leak prevention the tutor's number-one requirement for this course; it outranks every other improvement.
