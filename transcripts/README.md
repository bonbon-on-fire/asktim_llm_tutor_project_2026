# Transcripts

Generated tutor-student conversation transcripts for the AskTim LLM Tutor project.

## Folder Structure

Current on-disk layout. Each persona folder currently has exactly two
subfolders — `{persona}_raw/` (ungraded) and `{persona}_claude/`
(Claude-graded) — with **108 transcripts each** (324 raw + 324 graded).
The whole current corpus is a single course, `supply_chain_design`
(MIT CTL.SC2x), generated as "with-lecture-context" simulations with
`tutor_prompt=tutor_05`.

```text
transcripts/
├── chaotic/                          # Chaotic student persona
│   ├── chaotic_raw/                  # Raw (ungraded) transcripts — standard pipeline (108)
│   └── chaotic_claude/               # Claude-graded copies of chaotic_raw/ (108)
├── cooperative/                      # Cooperative student persona
│   ├── cooperative_raw/              # Raw transcripts — standard pipeline (108)
│   └── cooperative_claude/           # Claude-graded copies of cooperative_raw/ (108)
├── clueless/                         # Clueless student persona
│   ├── clueless_raw/                 # Raw transcripts — standard pipeline (108)
│   └── clueless_claude/              # Claude-graded copies of clueless_raw/ (108)
└── README.md
```

> The tooling is still general/multi-course and historically produced other
> suffix folders (`_gpt/`, `_mini/`, `_claude_mini/`, `_raw_tutor_05/`,
> `_claude_tutor_05/`) on demand via the judge/mini runners. None of those are
> present on disk right now — only `_raw/` and `_claude/` exist. The suffix
> conventions below document those historical variants for reference.

## Subfolder Convention

Each persona folder uses a suffix pattern to indicate the generation and grading
method. Only the first two suffixes below are present on disk today; the rest are
historical variants the tooling can still produce on demand.

Currently present:

- **`{persona}_raw/`** — Ungraded transcripts generated by the standard tutor+student simulation pipeline. Source-of-truth inputs to the judge.
- **`{persona}_claude/`** — Copies of raw transcripts with a `grade` object added by the Claude judge (`judge/run_judge.py`, provider=`claude`).

Historical variants (not currently on disk):

- **`{persona}_gpt/`** — Copies of raw transcripts graded by the GPT judge (`judge/run_judge.py`, provider=`gpt`).
- **`{persona}_mini/`** — Mini-continuation transcripts: a raw transcript forked at a pivot turn, with the student side replayed from file and the tutor regenerated using a new prompt or provider. Includes a `mini_continuation` metadata object. Output stem matches the source raw transcript stem.
- **`{persona}_claude_mini/`** — Claude-graded copies of `{persona}_mini/` transcripts.
- **`{persona}_raw_tutor_05/`** — Ungraded transcripts generated using `tutor_05` as the tutor prompt (produced with `internal_ui.run_ui_raw --output-suffix raw_tutor_05`).
- **`{persona}_claude_tutor_05/`** — Claude-graded copies of `{persona}_raw_tutor_05/` transcripts.

## Transcript JSON Schema

Every transcript file (`transcript_NNNN.json`) follows this structure:

```json
{
  "tutor_prompt": "tutor_03",
  "student_persona": "chaotic_01",
  "course": "cities_and_climate_change",
  "exercise_number": "01",
  "figures": [],
  "turn_size": 10,
  "context": "Course description text...",
  "exercise": "Full exercise prompt text...",
  "turns": 10,
  "exchanges": [
    {
      "turn": 1,
      "student": "Student message...",
      "tutor": "Tutor response...",
      "pedagogical_reasoning": "Internal tutor reasoning..."
    }
  ]
}
```

Graded transcripts additionally contain a top-level `grade` object:

```json
{
  "grade": {
    "sections": {
      "1_pedagogy": { "criteria": { "1.1": { ... }, "1.2": { ... }, "1.3": { ... } }, "base": { "score": 18, "max": 20 } },
      "2_dialogue_quality": { "criteria": { "2.1": { ... }, "2.2": { ... } }, "base": { "score": 11, "max": 12 } },
      "3_communication_quality": { "criteria": { "3.1": { ... }, "3.2": { ... } }, "base": { "score": 8, "max": 8 } }
    },
    "max_score": 40,
    "total_base_score": 37,
    "max_base_score": 40,
    "model": { "provider": "anthropic", "model": "claude-sonnet-4-6" },
    "overview": ["Brief evidence-based overview."],
    "total_score": 37,
    "judge_llm_calls": 1
  }
}
```

### Key Fields

| Field | Description |
| ----- | ----------- |
| `tutor_prompt` | Tutor system prompt version used |
| `student_persona` | Student persona identifier (e.g. `chaotic_01`, `cooperative_02`) |
| `course` | Course folder name under `curriculum/` (e.g. `cities_and_climate_change`, `mathematics_for_cs`, `meaning_of_life`, `physics_iii_vibrations_and_waves`, `intro_to_international_development_planning`) |
| `exercise_number` | Exercise identifier within the course |
| `figures` | Filenames of curriculum figures (under `curriculum/<course>/figures/`) attached to this run as multimodal input; `[]` when the exercise has none. The judge re-resolves these names to re-attach the same images at grading time. Absent on pre-June-2026 transcripts (treated as `[]`). |
| `turn_size` | Planned number of exchanges |
| `turns` | Actual number of exchanges |
| `exchanges` | Array of student-tutor exchange objects |
| `grade.total_score` | Total points awarded by the judge |
| `grade.max_score` | Maximum possible score (40 for the current `rubric_08`; 46 for the older `rubric_05`) |

## Student Personas

| Persona | Behavior |
| ------- | -------- |
| **chaotic** | Challenges boundaries, tests edge cases, requests direct answers |
| **cooperative** | Good-student baseline that stays on-task and follows guidance |
| **clueless** | Genuinely confused, needs scaffolding and patience |

Each persona has multiple versions (e.g. `chaotic_01`, `chaotic_02`).

## Transcript Naming

Files are named `transcript_NNN.json` where `NNN` is a sequential index. The
current corpus is zero-padded to **three digits** (`transcript_001` …
`transcript_108`). Each persona-type `_raw/` folder numbers independently from
`001`. The same index across `_raw` and `_claude` folders corresponds to the
same conversation — the graded version is a copy of the raw file with a `grade`
object appended.

> Note: older transcripts from before June 2026 used two- or four-digit padding
> (`transcript_01`, `transcript_0001`); such forms may coexist in a folder.
> Numbering is set by
> `_next_transcript_number()` in [`internal_ui/run_ui_raw.py`](../internal_ui/run_ui_raw.py).

## How Transcripts Are Generated

Raw transcripts are produced by the tutor-student simulation pipeline:

1. Select a tutor prompt, student persona, course, and exercise.
2. Run a multi-turn conversation (default 10 exchanges).
3. Save the transcript JSON to `{persona}/{persona}_raw/`.

## How Transcripts Are Graded

```powershell
# Grade all raw transcripts with Claude (current corpus)
python -m internal_ui.run_ui_judge --provider claude

# GPT grading is still supported by the tooling but is not part of the
# current on-disk corpus:
python -m internal_ui.run_ui_judge --provider gpt
```

Both commands accept `--prompt` and `--rubric` flags. Output goes to the
respective `_claude` or `_gpt` subfolder. The current corpus was graded by
`claude-sonnet-4-6` against `rubric_08` (40 points).

## Visualization

Grading charts are generated by the visualization module:

```powershell
# Score histogram + per-transcript line charts (all personas + one per family)
python -m visualization.run_visualization

# SC2x persona-type charts (bar / heatmap / boxplot by persona and problem)
python -m visualization.sc2x_eval_charts
```

Outputs go to `visualization/outputs/` (and `visualization/outputs/sc2x/` for the
SC2x charts). See [`visualization/README.md`](../visualization/README.md) for the
full chart list.
