Problem 3. Bounded Asymptotics [15 points]

In this problem, we investigate the asymptotic relationship of functions under some conditions. You can assume that any function mentioned in the problem maps ℕ to ℝ+. You might find the following definitions useful:
1. A function f is bounded from below (respectively above) if there exists a real number B such that f(n) ≥ B (respectively f(n) ≤ B) for all n ∈ ℕ.
2. A function f is bounded away from zero if there exists a real number B > 0 such that f(n) ≥ B for all n ∈ ℕ.
3. A function f is strictly increasing (respectively decreasing) if for any a, b ∈ ℕ we have that a > b ⇒ f(a) > f(b) (respectively a > b ⇒ f(a) < f(b)).

(a) [5 pts] Let f be a function that is bounded from above and g be a function that is bounded away from zero. Carefully prove that f ∈ O(g).
    Hint: Careful! The limit lim_{n→∞} f(n)/g(n) might not exist, so your argument shouldn't use limits.
(b) [5 pts] Let f be a function that is bounded from above and g be a function that is bounded from below. Is it necessarily true that f ∈ O(g)?
(c) [5 pts] Let f be a function that is strictly decreasing and g be a function that is strictly increasing. Is it necessarily true that f ∈ O(g)? Is it necessarily true that f ∈ o(g)?


Problem 4. Karatsuba's Algorithm [10 points]

As we'll see soon, cryptography often requires arithmetic with large integers, possibly with hundreds or thousands of digits each! When adding two integers A and B each with n digits, the straightforward algorithm requires at most 2n single-digit additions. But the straightforward multiplication algorithm involves multiplying each digit of A with each digit of B, which is already Θ(n^2) operations. Karatsuba's Algorithm is a clever divide-and-conquer algorithm with (hopefully) faster asymptotic runtime.

Karatsuba's Algorithm can multiply two n-digit numbers using at most T(n) single-digit arithmetic operations, where T(n) is defined by the following recurrence:
    T(0) = 0,   T(1) = 1,   T(n) = 3·T(⌈n/2⌉) + 10n   for n ≥ 2.

(a) [1 pt] Compute T(2), T(3), and T(4).
(b) [4 pts] Use the Plug and Chug method to find an exact (not just asymptotic!) closed-form expression for T(n), assuming n is an exact power of 2. You do not need to prove your formula, but you should check it against T(1), T(2), and T(4).
    Note: We'd like you to use the Plug and Chug method specifically; solutions that arrive at a closed form using other techniques (or without explanation) will only receive partial credit.
(c) [4 pts] Karatsuba's algorithm works whether or not n is a power of 2, and the same recurrence applies, so let's find an approximate solution for T(n) in this more general case: carefully use the Master Theorem to prove an asymptotic bound of the form T(n) ∈ Θ(g(n)), for some closed-form function g(n).
(d) [1 pt] Is Karatsuba an asymptotic improvement over the straightforward Θ(n^2) algorithm? Why or why not?

Optional Note: Here's a rough description of Karatsuba's Algorithm, in case you're curious. If A and B are n-digit numbers, we can split each list of digits in half by writing A = A0·X + A1 and B = B0·X + B1, where X = 10^⌈n/2⌉. We could compute A · B using the formula
    A · B = [A0·B0]·X^2 + ([A0·B1] + [A1·B0])·X + [A1·B1],
which requires 4 multiplications of ⌈n/2⌉-digit numbers, leading to T(n) = 4·T(⌈n/2⌉) + O(n) with solution Θ(n^2). Karatsuba cleverly observed that P, Q, R can be computed with just three multiplications:
    P = [A0·B0]
    R = [A1·B1]
    Q = P + R − [(A0 − A1)·(B0 − B1)].
This reduces the 4 to a 3 in the recurrence, while the O(n) term remains O(n), which directly leads to the asymptotic speedup.
